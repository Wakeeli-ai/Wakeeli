================================================================================
CRITICAL BUGS (500 Errors):
================================================================================

BUG-1: "not sure" after listings triggers 500 (Scenario 21)
  - Trigger: User says "not sure" after receiving property listings
  - Expected: Bot asks what they're unsure about
  - Impact: HIGH - very common user response

BUG-2: Booking confirmation triggers 500 (Scenario 24)
  - Trigger: User confirms a suggested visit time
  - Expected: Booking confirmation with property, date, agent handoff
  - Impact: CRITICAL - breaks the core booking flow

BUG-3: Reschedule request after booking triggers 500 (Scenario 26)
  - Trigger: User asks to reschedule after a booking is confirmed
  - Expected: Route to agent
  - Impact: HIGH - affects post-booking flow

BUG-4: Intermittent 500s on single-word drip messages (Scenario 28)
  - Trigger: Short single-word messages like "Achrafieh", "2 beds", name
  - Pattern: First attempt fails, retry succeeds, but state is corrupted
  - Impact: MEDIUM - intermittent but causes confusion on retry

================================================================================
BEHAVIORAL ISSUES (Non-Crash):
================================================================================

ISSUE-1: Prompt self-correction in responses (Scenarios 23, 24, 26, 27)
  - Bot outputs its "thinking" mid-response: "Wait, I already have all that
    info. Let me restart properly."
  - This internal reasoning should not be visible to users
  - Affects: Any message that bundles multiple info fields at once

ISSUE-2: Budget loop with no exit (Scenario 30)
  - Bot asks for budget repeatedly after user refuses 3+ times
  - No fallback path to show listings without budget
  - User is stuck in a loop with no way forward

ISSUE-3: Villa property type not recognized (Scenario 29)
  - Bot treats villa request same as apartment request
  - No acknowledgment of property type difference
  - Shows apartment listing for villa seeker without explanation

ISSUE-4: Listing count inconsistency (Scenario 27 vs 21)
  - Scenario 21 (budget $1,500): Showed 3 listings including above-budget ones
  - Scenario 27 (budget $5,000): Showed only 1 listing
  - Logic for how many listings to surface appears inconsistent

ISSUE-5: State corruption on 500 retry (Scenario 28)
  - When a 500 occurs, the message is processed server-side but response fails
  - On retry, bot acts confused ("Did you mean something else, or confirming?")
  - Messages should be idempotent or transactional
