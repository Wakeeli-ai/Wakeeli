================================================

1. [S22/S27] Name input hijacks flow: When bot is waiting for missing info and user sends their name, bot treats name as signal to show listings even if bedrooms/furnished preference are still missing. Budget confirmation is skipped.

2. [S26] No returning customer handling: Bot has no mechanism to recognize or acknowledge a returning lead. Treats every conversation as new.

3. [S27] Budget clarification loop: After bot correctly questions the $50,000 budget, user sends name instead of confirming budget. Bot skips budget confirmation entirely and shows listings.

4. [S28] Ambiguous bedroom request not handled: "Studio or 1-bedroom" should be treated as two valid options to search across, not a reason to re-ask for bedroom count.

5. [S29] Initial listing failure: Full-info message with valid Ashrafieh 1BR $800 led to agent escalation instead of listings. Possibly a no-results edge case that triggers agent fallback too quickly.
